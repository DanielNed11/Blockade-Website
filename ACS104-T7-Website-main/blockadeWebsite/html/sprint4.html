Data # 6: Wrap up and track changes - Hildo

Overview

This deliverable summarizes all the changes made after the hand-ins of Sprint 1, Sprint 2, and Sprint 3 for the Data & AI part of our project.

Chapter 1: Sprint 1

No changes were required for Sprint 1. All initial deliverables such as the database schema, DDL/DML scripts, and data model were complete and functional, so no updates were necessary.

Chapter 2: Sprint 2

Several updates were made to reflect the implementation of game statistics and leaderboard functionality:

  • New SQL functions were added to the database to calculate scores, play time, and move durations.
  • Move data is now stored for each game, enabling detailed statistics per player and per game.
  • The leaderboard was implemented in JavaFX using SQL queries and is updated in real time based on game results.
  • A visual chart (bar chart) was added to display the time taken per move, including outliers.

Chapter 3: Sprint 3

We made changes to support our rule-based AI player:

  • Java classes RuleBasedStrategy, InferenceEngine, and FactHandler were created to implement a simple AI using if-then rules.
  • The AI logic was integrated into the game and can now play against human players with basic decision-making.
  • Tile.java was updated to support vertical stacking of pieces, enabling better handling of AI moves.
